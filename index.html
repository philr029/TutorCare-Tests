            📖 Documentation
          

          
How to use the dashboard, configure API keys, and understand results.

        


        


          
          

            

              🚀 How to Use the Dashboard
            

            
    
              
  1. Configure the API endpoint — go to ⚙️ Settings, enter the base URL of your running FastAPI backend (e.g. https://api.example.com) and optionally your API key, then click Save.
    2. 
              
  2. Choose a tool from the sidebar on the left.
    3. 
              
  3. Enter the target (IP address, domain, phone number, or URL) into the form.
    4. 
              
  4. Click the action button or press Enter.
    5. 
              
  5. Review results — colour-coded badges, tables, and charts display the findings. Expand Raw JSON Response to see the full API payload.
    6. 
            

          


          
          

            

              🔑 How to Configure API Keys
            

            

              The dashboard itself does not require any external API keys. It communicates with the
              FastAPI backend, which optionally calls AbuseIPDB, VirusTotal, Numverify, and Twilio
              for enriched results.
            

            

              Set the following environment variables on the API server (or in its .env file):
            

            
ABUSEIPDB_KEY=your_abuseipdb_key
VIRUSTOTAL_KEY=your_virustotal_key
NUMVERIFY_KEY=your_numverify_key
TWILIO_SID=your_twilio_sid
TWILIO_TOKEN=your_twilio_token

# API authentication (comma-separated keys, or leave empty to disable)
API_KEYS=your-dashboard-key-here

            

              All keys are optional. Without them the toolkit still performs DNSBL lookups
              and offline phone-number parsing.

This toolkit is intended for authorized security research, system administration, and network diagnostics only.

              
Do not use this tool against systems you do not own or for which you do not have explicit written permission to test.

              
All checks performed are non-intrusive and passive:

              
    
                
  • DNS queries use standard resolution
    • 
                
  • HTTP requests use a clearly identified User-Agent
